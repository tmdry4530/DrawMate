## 11. E2E 시나리오

### 시나리오 A: 구직자 플로우

```
1. 회원가입 (/sign-up)
2. 온보딩: "어시스턴트" 역할 선택 + 프로필 작성
3. 스튜디오 진입 (/studio)
4. 새 포트폴리오 생성 (템플릿 선택 → 이미지 3장 업로드 → 정보 입력 → 태그 설정)
5. 임시 저장 후 재편집
6. 게시하기 → published 확인
7. 탐색 페이지에서 내 포트폴리오 검색 가능 확인
8. 구인자로부터 메시지 수신 → 알림 확인 → 대화방 응답
```

- [ ] 전체 플로우 막힘 없이 완료
- [ ] 각 단계 전환 시 데이터 유지

### 시나리오 B: 구인자 플로우

```
1. 회원가입 (/sign-up)
2. 온보딩: "작가/PD" 역할 선택 + 프로필 작성
3. 홈에서 포트폴리오 둘러보기
4. 탐색 (/explore) → 태그 필터 (웹툰 + 채색) → 결과 확인
5. 포트폴리오 상세 열람
6. 북마크 추가
7. "메시지 보내기" → 대화 시작 → 메시지 전송
8. 북마크 목록에서 재확인
9. 대화 목록에서 응답 확인
```

- [ ] 전체 플로우 막힘 없이 완료
- [ ] 검색 → 상세 → 연락 전환이 3클릭 이내

### 시나리오 C: 비관여 방문자

```
1. 비로그인 상태로 홈 접속
2. 탐색 페이지에서 포트폴리오 둘러보기
3. 포트폴리오 상세 열람
4. 작성자 프로필 확인
5. "메시지 보내기" 클릭 → 로그인 요구
6. 회원가입 후 이전 포트폴리오로 돌아와서 연락
```

- [ ] 비로그인 탐색이 자연스러움
- [ ] 인증 요구 시점이 적절함 (연락/북마크 시)
