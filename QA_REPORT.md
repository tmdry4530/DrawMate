# DrawMate 이슈 재테스트 리포트

**테스트 날짜:** 2026-03-23
**이전 테스트 대비 재검증 시점:** 동일 세션, 약 40분 경과 후

---

## 수정 완료 (6건)

### ✅ 이슈 #1 — `tagIds: []` PATCH 500 에러 (위저드 차단)

**이전:** `tagIds: []` 포함 PATCH → 500 "태그 동기화에 실패했습니다" → 위저드 Step 1→2 진행 불가
**현재:** PATCH 200 정상 반환. 위저드에서 템플릿 선택 → "다음" 클릭 시 Step 2(이미지 업로드) 정상 진입 확인. 자동 저장 "오전 05:35 저장됨" 표시. 진행률 "2/4 (25%)" 갱신 정상.

### ✅ 이슈 #2 — 북마크 API 500 에러

**이전:** POST `/api/v1/portfolios/{id}/bookmark` → 500
**현재:** POST → 201 `{bookmarked: true}`. UI에서도 하트 아이콘 채워짐 + 카운트 0→1 변경 확인. 재클릭 시 해제 + 카운트 1→0 복원 정상.

### ✅ 이슈 #3 — 메시지 전송 API 500 에러

**이전:** POST `/api/v1/conversations/{id}/messages` → 500
**현재:** POST → 201 성공. UI에서 직접 메시지 입력 → 전송 → 파란색 말풍선 즉시 표시, 입력 필드 초기화, 좌측 대화 목록에 최신 메시지 미리보기 갱신 모두 정상.

### ✅ 이슈 #4 — `/api/v1/me` 엔드포인트 404

**이전:** GET `/api/v1/me` → 404 → 설정 페이지 "설정 정보를 불러오지 못했습니다" 에러 토스트
**현재:** GET → 200 `{userId, profile, counts}` 정상 반환. 설정 페이지 에러 토스트 사라짐.

### ✅ 이슈 #6 — sitemap.xml / robots.txt 도메인 불일치

**이전:** 모든 URL이 `drawmate.kr`
**현재:** sitemap.xml 내 URL 전부 `drawmate.xyz`로 변경. robots.txt의 Sitemap 경로도 `https://drawmate.xyz/sitemap.xml`으로 수정 완료.

### ✅ 이슈 #7 — 공개 프로필 포트폴리오 그리드 미렌더링

**이전:** `/users/[userId]`에서 "2 작품" 표시되지만 카드 미노출
**현재:** 포트폴리오 카드 2개("1231", "asdas") 정상 렌더링.

### ✅ 이슈 #8 — OG 메타태그 미구현

**이전:** 포트폴리오 상세 페이지에 og 태그 없음
**현재:** `og:title`, `og:description`, `og:image`, `og:type` 모두 구현 확인.

## 미수정 잔여 이슈 (3건)

### ❌ 이슈 #5 — 탐색 필터: 단일 태그 클릭 시 카테고리 전체 선택 [P1]

**현상:** /explore에서 "웹툰" 체크박스 하나만 클릭하면 분야 카테고리 5개(웹툰, 일러스트, 애니메이션, 배경, 캐릭터)가 동시에 체크됨. 결과적으로 AND 조건으로 "검색 결과가 없습니다" 표시.
**영향:** 탐색 필터링 기능 사실상 사용 불가. 사용자가 특정 분야만 골라볼 수 없음.
**재현:** /explore → 분야 섹션 → 아무 체크박스 1개 클릭

### ❌ 이슈 #9 — `/explore?fieldTags[]=...` URL 파라미터 미반영 [P2]

**현상:** 홈 인기 태그 클릭 → `/explore?fieldTags[]=field-webtoon` 이동 시, URL에 필터 파라미터가 존재하지만 좌측 체크박스 UI에 반영되지 않음. 결과 화면은 빈 상태("검색 결과가 없습니다").
**영향:** 홈 → 탐색 태그 연동 불가.

### ❌ 이슈 #10 (Minor) — 태그 카테고리 라벨 영어 표시 [P3]

**현상:** 포트폴리오 상세 페이지 태그 섹션의 카테고리 라벨이 영어(`field`, `skill`, `tool`, `style`)로 표시.
**기대:** 한글 표시 (분야, 스킬, 도구, 스타일)

## 신규 발견 이슈 (2건)

### 🆕 이슈 #11 — 프로필 편집 폼 기존 데이터 미로딩 [P2]

**현상:** `/settings/profile` 진입 시 이름, 한 줄 소개 등 필드가 placeholder만 표시. `/api/v1/me` 응답에 `displayName: "asdasd"`가 포함되어 있으나 폼에 채워지지 않음.
**영향:** 사용자가 기존 프로필을 확인/수정할 수 없음. 빈 상태에서 저장하면 기존 데이터가 덮어씌워질 위험.

### 🆕 이슈 #12 — 대화 목록 상대방 이름 "알 수 없음" 표시 [P2]

**현상:** `/messages` 대화 목록 및 대화방 헤더에서 상대방 이름이 "알 수 없음"으로 표시. 실제 상대 사용자의 displayName은 "테스트1"로 존재함.
**영향:** 누구와 대화 중인지 식별 불가.
